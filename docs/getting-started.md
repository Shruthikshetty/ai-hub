# Getting Started with AI Hub

Welcome to **AI Hub**, your personal AI aggregator. AI Hub brings together the world's most powerful AI models—from OpenAI and Google to specialized providers like ElevenLabs—all into one unified, locally-stored interface.

> [!IMPORTANT]
> **Privacy First:** Your chats and keys are stored locally. Read our [Security & Privacy Guide](./security.md) to understand how your data is handled.

## 🚀 Quick Start Guide

Getting up and running takes just a few minutes:

### 1. Obtain an API Key
To use the models, you'll need an API key from a provider. 
*   **Pro Tip:** For a quick, free tryout, we recommend **Google AI Studio**. You can get a free API key there and start chatting immediately.
*   **OpenAI Standard:** Most providers in AI Hub follow the OpenAI specification, ensuring a stable and reliable experience.

### 2. Configure your Provider
1.  Open the app and head to the **Settings** tab.
2.  Select the **Providers** section.
3.  Find your provider (e.g., Google or OpenAI), enter your API key, and toggle it to **Enabled**.
4.  Save your settings.

### 3. Start Chatting & Generating
Once configured, you can head to the main interface to:
*   **Chat:** Interact with LLMs for text generation and reasoning.
*   **Generate Images/Video:** Use specialized models to create media.
*   **TTS (Text-to-Speech):** Convert your text into high-quality audio.

> [!TIP]
> Always check the [Provider Matrix](./provider-matrix.md) to see exactly which features (Chat, Image, Video, etc.) are supported by each provider.

## 🛠️ Advanced Features & Options

### The Side Tray
For more control over your experience, look for the **Side Tray**. This is where you'll find advanced options such as:
*   **System Prompts:** Set the "personality" or instructions for the AI.
*   **Tool Selection:** Enable features like **Web Search**.
    *   *Note:* Some providers only support search using their own built-in tools.

---

## 🔍 Troubleshooting & Model Behavior

Because AI Hub is an aggregator that "dumps" many models together into one interface, you may occasionally encounter issues:
*   **Non-Responsive Models:** Not all models support all input types or protocols. If a model fails to respond, it is often due to provider-side limitations.
*   **What to do?** If a model isn't working as expected, try:
    1.  Switching to a different **Model** within the same provider.
    2.  Checking your **Advanced Options** in the Side Tray.
    3.  Verifying the provider's status in the [Provider Matrix](./provider-matrix.md).

Happy Chatting!
